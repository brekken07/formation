📦 Init & Format
terraform init
Initialise le répertoire, télécharge providers & modules
terraform init -upgrade
Met à jour les providers selon les contraintes
terraform init -migrate-state
Migre le state vers un nouveau backend
terraform init -reconfigure
Reconfigure le backend sans migrer le state
terraform fmt
Formate les fichiers .tf selon le style canonique
terraform fmt -recursive
Formate aussi les sous-répertoires
terraform fmt -check
Vérifie le format (exit 1 si non formaté)
🔍 Validate & Plan
terraform validate
Valide la syntaxe sans accès réseau
terraform plan
Prévisualise les changements (dry-run)
terraform plan -out=tfplan
Sauvegarde le plan dans un fichier binaire
terraform plan -destroy
Prévisualise une destruction complète
terraform plan -var-file=prod.tfvars
Utilise un fichier de variables spécifique
terraform plan -var "key=val"
Passe une variable en ligne de commande
terraform plan -target=res.name
Limite le plan à une ressource cible
✅ Apply & Destroy
terraform apply
Applique le plan (demande confirmation)
terraform apply tfplan
Applique exactement le plan sauvegardé
terraform apply -auto-approve
Applique sans confirmation (CI/CD)
terraform apply -replace=res.name
Force la recréation d'une ressource
terraform apply -refresh-only
Met à jour le state sans modifier l'infra
terraform destroy
Détruit toutes les ressources du state
terraform destroy -target=res.name
Détruit uniquement la ressource ciblée

Blocs HCL — Structure de configuration
terraform {} & provider {}
HCL terraform { required_version = ">= 1.5" required_providers { aws = { source = "hashicorp/aws" version = "~> 5.0" } } backend "s3" { bucket = "my-tf-state" key = "prod/terraform.tfstate" region = "eu-west-1" dynamodb_table = "tf-locks" encrypt = true } } provider "aws" { region = "eu-west-1" } # Provider avec alias (multi-région) provider "aws" { alias = "us" region = "us-east-1" }
resource {} & data {}
HCL # resource = créer/gérer l'infra resource "aws_instance" "web" { ami = data.aws_ami.ubuntu.id instance_type = var.instance_type tags = local.common_tags # Référence alias provider provider = aws.us lifecycle { create_before_destroy = true prevent_destroy = false ignore_changes = [tags] } } # data = lire une ressource existante data "aws_ami" "ubuntu" { most_recent = true owners = ["099720109477"] filter { name = "name" values = ["ubuntu/images/*22.04*"] } } # Référence : data.aws_ami.ubuntu.id
variable {} & locals {} & output {}
HCL variable "instance_type" { type = string default = "t3.micro" description = "Type d'instance EC2" sensitive = false validation { condition = contains( ["t3.micro","t3.small"], var.instance_type) error_message = "Type non autorisé." } } locals { env = terraform.workspace common_tags = { Env = local.env ManagedBy = "Terraform" } } output "instance_ip" { value = aws_instance.web.public_ip description = "IP publique de l'instance" sensitive = false }
module {} & moved {}
HCL # Appel d'un module Registry module "vpc" { source = "terraform-aws-modules/vpc/aws" version = "~> 5.0" name = "my-vpc" cidr = var.vpc_cidr } # Accès : module.vpc.vpc_id # Module local module "sg" { source = "./modules/security_group" vpc_id = module.vpc.vpc_id } # Refactoring sans destroy/recreate moved { from = aws_instance.old_name to = aws_instance.new_name } # depends_on explicite resource "aws_instance" "app" { depends_on = [aws_iam_role_policy.attach] }

Meta-arguments & Lifecycle
count & for_each
# count — basé sur un entier resource "aws_instance" "web" { count = var.create ? 1 : 0 ami = var.ami } # Référence : aws_instance.web[0] # for_each — basé sur map/set resource "aws_instance" "nodes" { for_each = var.instances ami = each.value.ami instance_type = each.value.type tags = { Name = each.key } } # Référence : aws_instance.nodes["prod"]
lifecycle
lifecycle { # Créer avant détruire (zero-downtime) create_before_destroy = true # Bloque toute destruction prevent_destroy = true # Ignore les changements sur ces attrs ignore_changes = [ tags, instance_type, ] # Condition de remplacement replace_triggered_by = [ aws_launch_template.this.id, ] } # Remplacer une ressource (anciennement taint) # terraform apply -replace=aws_instance.web
dynamic {}
# Générer des blocs répétitifs resource "aws_security_group" "web" { name = "web-sg" vpc_id = var.vpc_id dynamic "ingress" { for_each = var.ingress_rules content { from_port = ingress.value.from to_port = ingress.value.to protocol = ingress.value.proto cidr_blocks = ingress.value.cidrs } } } # ingress.key / ingress.value disponibles

State & Backends
Backend S3 + DynamoDB (AWS — recommandé)
terraform { backend "s3" { bucket = "my-tf-state-bucket" key = "env/prod/terraform.tfstate" region = "eu-west-1" dynamodb_table = "terraform-state-lock" encrypt = true kms_key_id = "alias/terraform-state" } } # Table DynamoDB requise : # - Primary key : LockID (type String) # - S3 bucket : versionning activé # - Partial config via : # terraform init -backend-config=secret.hcl
💡
S3 = stockage & versioning · DynamoDB = verrou · KMS = chiffrement

Pièges HCTA-003 & Points critiques
🔄 Init & Lock
terraform get
≠ terraform init : get télécharge uniquement les modules, pas les providers ni le backend
-upgrade
Nécessaire pour mettre à jour les providers au-delà du lock file
.terraform.lock.hcl
À commiter dans Git. Ne jamais ignorer ce fichier
📊 State
state rm
Supprime du state SANS détruire l'infra réelle
sensitive=true
Cache de l'output CLI, mais le state reste en clair
apply partiel
Si apply échoue à mi-chemin, le state reflète ce qui a réussi (pas de rollback)
🧩 Modules
TF_VAR_
Ne se propagent PAS aux modules enfants
terraform.tfvars
Lu seulement par le module racine
backend dans module enfant
INTERDIT — le backend est uniquement dans le module racine
⚡ Expressions
for_each + list
for_each nécessite un map ou set, pas une liste directement
count vs for_each
count[0] peut décaler les index ; for_each est stable par clé
if/else
N'existe pas en HCL — utiliser l'opérateur ternaire ? :
