Inventory — Hôtes & Groupes
Inventory INI (classique)
INI# Hôtes individuels web1.example.com web2.example.com ansible_port=2222 # Groupe [webservers] web1.example.com web2.example.com web[3:9].example.com # pattern [dbservers] db1.example.com ansible_user=postgres # Groupe de groupes [prod:children] webservers dbservers # Variables de groupe dans l'inventory [webservers:vars] ansible_user=ubuntu ansible_python_interpreter=/usr/bin/python3
Inventory YAML (moderne) & Variables hôte
YAML# inventory.yml all: children: webservers: hosts: web1: ansible_host: 192.168.1.10 ansible_user: ubuntu web2: ansible_host: 192.168.1.11 vars: http_port: 80 dbservers: hosts: db1: ansible_host: 192.168.1.20 vars: ansible_ssh_private_key_file: ~/.ssh/id_rsa

Préférer les répertoires host_vars/ et group_vars/ pour séparer les variables de l'inventory.

Playbook — Structure complète
Playbook complet avec toutes les directives
YAML--- # playbook.yml - name: Deploy web application hosts: webservers gather_facts: true # collecte les facts become: true # sudo global become_user: root serial: 2 # rolling update: 2 à la fois max_fail_percentage: 30 # tolère 30% d'erreurs any_errors_fatal: false timeout: 30 tags: deploy vars: app_version: "1.2.3" app_port: 8080 vars_files: - vars/common.yml - vars/{{ ansible_os_family }}.yml pre_tasks: - name: Vérifier la version OS ansible.builtin.assert: that: ansible_distribution_major_version | int >= 20 roles: - common - nginx tasks: - name: Déployer l'application ansible.builtin.template: src: app.conf.j2 dest: /etc/app/app.conf mode: '0644' notify: restart app tags: [config] post_tasks: - name: Vérification finale ansible.builtin.uri: url: "http://localhost:{{ app_port }}/health" status_code: 200 handlers: - name: restart app ansible.builtin.service: name: myapp state: restarted
Contrôle de flux dans les tâches
YAML# Conditions - name: Installer sur Debian/Ubuntu ansible.builtin.apt: name: nginx state: present when: ansible_os_family == "Debian" - name: Installer sur RedHat ansible.builtin.yum: name: nginx when: - ansible_os_family == "RedHat" - ansible_distribution_major_version | int >= 8 # Boucle avec loop - name: Créer des utilisateurs ansible.builtin.user: name: "{{ item.name }}" groups: "{{ item.groups }}" loop: - { name: alice, groups: sudo } - { name: bob, groups: docker } # Register + debug - name: Vérifier nginx ansible.builtin.command: nginx -v register: nginx_version changed_when: false - ansible.builtin.debug: msg: "Version: {{ nginx_version.stdout }}" when: nginx_version.rc == 0 # Gestion d'erreurs - name: Tâche qui peut échouer ansible.builtin.command: /opt/script.sh ignore_errors: true failed_when: result.rc not in [0, 1] retries: 3 delay: 5 until: result.rc == 0 # Blocks - block: - name: Tâche risquée ansible.builtin.command: /risky.sh rescue: - name: En cas d'erreur ansible.builtin.debug: msg: "Échec — rollback" always: - name: Toujours exécuté ansible.builtin.debug: msg: "Nettoyage terminé"

Rôles — Structure & Bonnes pratiques
Structure d'un rôle Ansible
FSroles/ └── nginx/ ├── tasks/ │ ├── main.yml # entrée principale │ └── install.yml # importé depuis main ├── handlers/ │ └── main.yml # handlers du rôle ├── templates/ │ └── nginx.conf.j2 # templates Jinja2 ├── files/ │ └── index.html # fichiers statiques ├── vars/ │ └── main.yml # variables haute priorité ├── defaults/ │ └── main.yml # valeurs par défaut (basse priorité) ├── meta/ │ └── main.yml # dépendances du rôle └── README.md # Créer automatiquement la structure : $ ansible-galaxy role init nginx
requirements.yml & Appel de rôle
YAML# requirements.yml --- roles: - name: geerlingguy.nginx - src: https://github.com/org/role version: v2.0.0 name: my_nginx collections: - name: community.docker version: ">=3.0" # Installer : $ ansible-galaxy install -r requirements.yml --- # Appel de rôle dans un playbook - hosts: webservers roles: - common - role: nginx vars: nginx_port: 8080 tags: [web] when: install_nginx | bool # Ou via include/import : - ansible.builtin.include_role: name: nginx

Pièges fréquents & Bonnes pratiques
⚠️ command vs shell
# command : pas de shell # ❌ pipes, redirections, &&, || - ansible.builtin.command: cmd: "echo hello > /tmp/f" # ✗ Ne fonctionne PAS # shell : avec shell bash # ✅ pipes et redirections OK - ansible.builtin.shell: cmd: "ps aux | grep nginx" # Préférer les modules natifs # ✅ ansible.builtin.copy # ✅ ansible.builtin.apt # ✅ ansible.builtin.service # au lieu de shell/command
⚠️ Idempotence
# ❌ Pas idempotent - ansible.builtin.shell: cmd: "useradd myuser" # Échoue si déjà créé # ✅ Idempotent - ansible.builtin.user: name: myuser state: present # Contrôle changed_when - ansible.builtin.command: cmd: nginx -t changed_when: false # Ne signale jamais de changement
✅ Handlers
# Handlers : exécutés UNE seule fois # à la FIN du play, si notifiés tasks: - ansible.builtin.template: src: nginx.conf.j2 dest: /etc/nginx/nginx.conf notify: restart nginx handlers: - name: restart nginx ansible.builtin.service: name: nginx state: restarted # Forcer les handlers immédiatement - ansible.builtin.meta: flush_handlers
✅ include vs import
# import_* = statique (au chargement) # ✅ Tags et conditions s'appliquent - ansible.builtin.import_tasks: file: install.yml - ansible.builtin.import_role: name: nginx # include_* = dynamique (à l'exécution) # ✅ Peut utiliser des variables - ansible.builtin.include_tasks: file: "{{ ansible_os_family }}.yml" - ansible.builtin.include_role: name: "{{ role_name }}" # Règle : import pour structure fixe # include pour contenu dynamique
