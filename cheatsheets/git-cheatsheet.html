Résolution de conflits
Conflit# Git marque le fichier : <<<<<<< HEAD version actuelle (branche courante) ======= version entrante (branche mergée) >>>>>>> feature/login
git status
1. Voir les fichiers en conflit
git add fichier-conflit.py
3. Après résolution manuelle : stager
git merge --continue
4. Continuer le merge
git merge --abort
Annuler et revenir à l'état avant merge
git merge
git merge feature/login
Fusionne feature dans la branche courante
git merge --no-ff feature
Force un commit de merge (traçabilité)
git merge --squash feature
Écrase tous les commits de feature en 1 seul
git merge --abort
Annuler un merge en cours
💡
fast-forward = pas de commit de merge si la branche n'a pas divergé. --no-ff force toujours un commit de merge.
git rebase
git rebase main
Rejoue les commits de la branche courante sur main
git rebase -i HEAD~3
Rebase interactif — nettoyer les 3 derniers commits
git rebase --abort
Annuler un rebase en cours
git rebase --continue
Continuer après résolution de conflit
🚨
Ne jamais rebaser une branche déjà partagée/pushée. En rebase interactif : pick, squash, reword, drop.

🔬 git bisect — Trouver un bug par dichotomie
git bisect start
1. Démarrer la recherche dichotomique
git bisect bad
2. Marquer le commit courant comme buggué
git bisect good v1.2.0
3. Marquer un commit connu comme bon
git bisect good / bad
4. Tester + marquer — Git se déplace au milieu
git bisect reset
5. Terminer et revenir à HEAD
git bisect run ./test.sh
Automatiser avec un script de test

Annuler & Corriger
🧭 Quelle commande choisir ?
SituationCommande recommandéeImpact
Fichier modifié, pas stagégit restore fichiersafe
Fichier stagé (git add fait)git restore --staged fichiersafe
Dernier commit (pas pushé)git commit --amend ou git reset HEAD~1local
Plusieurs commits (pas pushés)git reset HEAD~Nlocal
Commit déjà pushé / partagégit revert <sha>safe
Fichiers non suivis à supprimergit clean -fdirréversible
Tout perdu après un reset hardgit reflog + git checkout SHArécupérable
git restore
git restore fichier.py
Annule les modifs non stagées
git restore --staged fichier.py
Retire du staging (unstage)
git restore --source HEAD~2 fichier.py
Restaure depuis un commit
git restore .
Annule toutes les modifs non stagées
git reset
git reset --soft HEAD~1
Annule commit → garde staging
git reset HEAD~1
Annule commit + staging → garde fichiers
git reset --hard HEAD~1
⚠ Tout effacer (commit + modifs)
git reset --hard origin/main
Revenir à l'état du remote
🚨
--hard est irréversible sans reflog. Ne jamais reset des commits partagés.
git revert & clean
git revert HEAD
Crée un commit d'annulation du dernier commit
git revert a3f9c21
Annule un commit spécifique de façon sûre
git revert --no-edit HEAD
Revert sans ouvrir l'éditeur
git clean -n
Prévisualiser ce qui serait supprimé
git clean -fd
Supprimer fichiers et dossiers non suivis
git reflog — Le filet de sécurité
git reflog
Historique complet des déplacements de HEAD
git checkout HEAD@{3}
Revenir à un état passé via le reflog
git branch sauvetage b72d1e8
Créer une branche depuis un commit "perdu"
✅
Le reflog est local, conservé 90 jours par défaut. Même un reset --hard est récupérable via reflog.

Stash — Mise en attente temporaire
Commandes Stash
git stash
Met de côté toutes les modifications en cours
git stash push -m "WIP: auth module"
Stash avec un message descriptif
git stash -u
Inclure les fichiers non suivis
git stash list
Lister tous les stashs
git stash pop
Restaurer le dernier stash et le supprimer
git stash apply stash@{1}
Appliquer un stash sans le supprimer
git stash show -p
Voir le contenu du dernier stash (diff)
git stash drop stash@{0}
Supprimer un stash spécifique
git stash clear
Vider tous les stashs
git stash branch nouvelle-branche
Créer une branche depuis un stash
Cas d'usage typique
Workflow# Tu travailles sur feature/A # Un bug urgent arrive sur main ! $ git stash Saved working directory and index state WIP $ git switch main $ git switch -c hotfix/urgent # Corriger le bug... $ git commit -m "fix: correctif urgent" $ git push # Revenir sur feature/A $ git switch feature/A $ git stash pop On branch feature/A: restored working state
💡
Le stash fonctionne comme une pile (LIFO). pop = apply + drop du dernier. apply = apply sans supprimer.

Semantic Versioning (SemVer)
SemVerMAJOR.MINOR.PATCH # MAJOR — breaking changes v2.0.0 → API incompatible # MINOR — nouvelles fonctionnalités compatibles v1.3.0 → feat: ajout endpoint /users # PATCH — correctifs compatibles v1.3.2 → fix: null pointer exception # Pre-release / build v1.0.0-alpha.1 v1.0.0-rc.2 v1.0.0+build.42
✅
Toujours utiliser les tags annotés pour les releases : ils embarquent auteur, date et message, contrairement aux tags légers.
