udev & périphériques
01 Surveiller les événements udev en temps réel ▼
À retenir
udevadm monitor est la sous-commande dédiée pour écouter les événements matériels en temps réel. Elle affiche les événements kernel ET udev au fur et à mesure. 
udevadm monitor              # kernel + udev
udevadm monitor --kernel      # kernel uniquement
udevadm monitor --udev        # udev uniquement
Piège
journalctl -f affiche les logs systemd en général — ce n'est pas spécifique aux événements udev.
Mémo sous-commandes udevadm
monitor · info · trigger · settle · test · control
02 Afficher les attributs udev en hiérarchie ▼
À retenir
udevadm info --attribute-walk affiche tous les attributs du device ET de ses devices parents, en hiérarchie. Indispensable pour écrire des règles udev. 
udevadm info --attribute-walk /dev/sda
udevadm info -a /dev/sda      # forme courte
Piège
--tree n'est pas une option valide de udevadm info.

Systèmes de fichiers
03 mkfs sans argument → quel type par défaut ? ▼
À retenir
Sans option -t, mkfs crée un système ext2 par défaut — pas ext3, pas ext4. 
mkfs /dev/sdb1               # → ext2 par défaut
mkfs -t ext4 /dev/sdb1       # → ext4 explicite
mkfs.ext4 /dev/sdb1          # → idem
Piège classique LPIC
ext2 ≠ ext3 ≠ ext4. Le défaut historique est ext2. Ne pas confondre avec le format le plus courant aujourd'hui (ext4).
04 Espace réservé par défaut sur ext4 : 5% ▼
À retenir
Par défaut, ext4 réserve 5% de l'espace total pour root, afin d'éviter les blocages système quand le disque est plein. 
tune2fs -m 2 /dev/sda1       # réduire à 2%
tune2fs -l /dev/sda1         # voir les paramètres
Piège
Les réponses fréquentes sont 5%, 10%, 3%. La bonne est 5%.
05 tune2fs — options à ne pas confondre ▼
OptionRôle
-i 200Intervalle de vérification en jours
-c 30Vérification tous les N montages
-m 5Pourcentage d'espace réservé pour root
-lLister les paramètres du fs
-CForcer le compteur de montages
Piège
Pour l'intervalle de vérification : c'est -i (interval), pas -d (qui n'existe pas) ni -c (count).

06 modprobe -r vs rmmod ▼
CommandeComportement

Mémo
modprobe = intelligent (gère les dépendances) — comme apt vs dpkg. rmmod = brut (module seul).
07 dmesg -C vs dmesg -c — vider le ring buffer ▼
OptionComportement
dmesg -CVide le ring buffer sans afficher (Clear only)
dmesg -cAffiche puis vide le ring buffer
dmesg --clearN'existe pas !
Piège
La casse change tout : -C majuscule ≠ -c minuscule. Et --clear n'existe pas.

Shell & traitement de texte
08 Redirections bash : 1>&2 vs 2>&1 ▼
NotationSens
2>&1stderr → stdout (le plus courant)
1>&2stdout → stderr
Mémo
Lecture : "fd source vers fd destination". Le & devant le chiffre indique un descripteur de fichier, pas un nom de fichier.

fd 0 = stdin · fd 1 = stdout · fd 2 = stderr 
# Envoyer stdout ET stderr dans un fichier :
commande > fichier.log 2>&1
# Ignorer les erreurs :
commande 2>/dev/null
09 grep -E et regex étendues — filtrer par heure ▼
À retenir
Pour matcher les logs entre 8h00 et 8h59, il faut cibler le champ heure avec :08: et utiliser -E pour les regex étendues. 
grep -E ':08:[0-9]+:[0-9]+' log.txt
# ERE = Extended Regular Expressions
# Permet : +  |  ()  sans échappement
Mémo
grep (BRE) · grep -E ou egrep (ERE) · grep -P (PCRE Perl)
10 source vs . vs exécution directe ▼
SyntaxeSous-shell créé ?
./script.shOui
/bin/bash script.shOui
source script.shNon — shell courant
. script.shNon — shell courant
Mémo
source et . sont équivalents. L'intérêt : les variables et exports persistent dans la session courante après exécution.
11 watch pour les fichiers /proc dynamiques ▼
À retenir
tail -f fonctionne uniquement pour les fichiers qui grossissent. Les fichiers /proc sont dynamiques (contenu rechargé en mémoire) → utiliser watch. 
watch -n 1 'cat /proc/interrupts'
watch -n 2 'cat /proc/meminfo'
watch -n 1 free -h
Mémo
watch -n N = rafraîchir toutes les N secondes. Par défaut : 2s.
12 tr -d '\r' — conversion Windows → Unix ▼
À retenir
Windows utilise \r\n (CR-LF). Unix utilise \n (LF). Pour supprimer le \r : 
tr -d '\r' < fichier_win.txt > fichier_unix.txt
# Alternatives :
sed 's/\r//' fichier.txt
dos2unix fichier.txt     # si installé
Mémo
Pour l'exam, la réponse canonique est tr -d '\r'. dos2unix peut ne pas être disponible.
13 find -maxdepth — la profondeur commence à 0 ▼
OptionPortée
-maxdepth 1Répertoire courant uniquement (profondeur 0)
-maxdepth 2Courant + sous-répertoires directs (0 et 1)
-maxdepth 3Courant + 2 niveaux en dessous
Piège
La profondeur du répertoire courant (.) est 0, pas 1. Donc pour inclure les sous-répertoires immédiats, il faut -maxdepth 2, pas 1.

Boot, init & initramfs
15 SysV init — /sbin/init et PID 1 ▼
À retenir
Le kernel lance /sbin/init comme premier processus (PID 1). Init reste actif toute la vie du système jusqu'au shutdown.
Piège
SysV init et systemd sont deux systèmes alternatifs, pas séquentiels. Sur un système SysV pur, init ne "passe jamais le relais" à systemd.
Mémo
SysV → /sbin/init
systemd → /lib/systemd/systemd (avec lien depuis /sbin/init)
Les deux ont toujours PID 1.
16 initramfs — ce que c'est et ce que ce n'est pas ▼
À retenir
L'initramfs est une archive cpio compressée chargée en RAM par GRUB. Elle contient les drivers et outils pour monter le vrai /. Ce n'est pas le kernel, ni le MBR.
ComposantEmplacement
MBR / bootloaderGéré par GRUB
Kernel/boot/vmlinuz-*
initramfs/boot/initrd.img-*
Contenu après boot/run/initramfs/
# Regénérer l'initramfs :
update-initramfs -u              # Debian/Ubuntu
dracut --force                   # RHEL/Fedora

Gestion des packages & dépôts
17 apt-get upgrade vs dist-upgrade vs apt full-upgrade ▼
CommandeComportement
apt-get upgradeMet à jour sans supprimer ni installer de nouveaux packages
apt-get dist-upgradeGère les dépendances complexes, peut supprimer des packages obsolètes
apt full-upgradeÉquivalent de dist-upgrade dans l'interface apt
Piège
full-upgrade existe dans apt, mais pas dans apt-get. Si la question porte sur apt-get, la réponse est dist-upgrade.
