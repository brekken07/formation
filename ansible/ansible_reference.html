1. Pourquoi Ansible ?

Ansible est un outil d'automatisation qui permet de gérer des machines Linux (et Windows) à distance, depuis une machine de contrôle, sans installer d'agent sur les cibles.

Le problème qu'il résout

Sans Ansible : connexion SSH manuelle sur chaque serveur, commandes répétées, erreurs humaines, aucune traçabilité.

Avec Ansible : on décrit l'état désiré dans un fichier texte, Ansible l'applique partout, de façon identique et reproductible.

Concept clé : l'idempotence

Comment ça fonctionne

Ansible s'installe uniquement sur la machine de contrôle.

Il se connecte aux machines cibles via SSH (pas d'agent à installer sur les cibles).

Il a besoin de Python 3 sur les machines cibles (présent par défaut sur Debian/Ubuntu modernes).

Structure recommandée

5. Modules essentiels

Un module = une action atomique. Ansible possède plus de 3000 modules. Voici les indispensables pour gérer une infrastructure Linux.

ansible.builtin.apt — gestion des paquets (Debian/Ubuntu)

ansible.builtin.service / systemd — gestion des services

ansible.builtin.user — gestion des utilisateurs

ansible.builtin.copy — copier des fichiers

ansible.builtin.template — déployer un fichier Jinja2

ansible.builtin.file — gérer fichiers, dossiers, liens

ansible.builtin.lineinfile — modifier une ligne dans un fichier

ansible.builtin.command et ansible.builtin.shell

ansible.builtin.debug — afficher des informations
