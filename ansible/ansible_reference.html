ℹ Idempotence

Lancer le même playbook 10 fois donne le même résultat. Ansible vérifie l'état actuel avant d'agir. changed: 1 = modification effectuée — changed: 0 = état déjà conforme.

02Installation
# Sur la machine de contrôle uniquement
$ pip3 install ansible          # via pip (recommandé)
$ sudo apt install ansible -y  # via dépôt Debian

$ ansible --version             # vérification
Prérequis sur les machines cibles
# Générer une clé SSH
$ ssh-keygen -t ed25519 -C "ansible" -f ~/.ssh/ansible_key

# Copier sur les cibles
$ ssh-copy-id -i ~/.ssh/ansible_key.pub user@IP_CIBLE

# Tester
$ ssh -i ~/.ssh/ansible_key user@IP_CIBLE
03Inventory
Structure recommandée
mon-projet/
├── inventory/
│   ├── hosts.ini            ← hôtes et groupes
│   └── group_vars/
│       ├── all.yml          ← vars communes à TOUS
│       └── webservers.yml   ← vars du groupe webservers
└── site.yml
hosts.ini — syntaxe complète
[webservers]
machine2   ansible_host=10.116.0.238
machine3   ansible_host=10.116.0.239

[databases]
db-01      ansible_host=10.116.0.240

# Groupe parent
[production:children]
webservers
databases

# Variables du groupe
[webservers:vars]
ansible_user=setupuser
ansible_ssh_private_key_file=~/.ssh/ansible_key
ansible_python_interpreter=/usr/bin/python3
Directives de connexion
DirectiveRôle
ansible_hostIP ou hostname réel
ansible_userUtilisateur SSH
ansible_portPort SSH (défaut : 22)
ansible_ssh_private_key_fileChemin vers la clé privée SSH
ansible_python_interpreterChemin vers Python sur la cible

group_vars/all.yml
---

formation_name: "Formation Ansible"

outils_base:
  - vim
  - curl
  - git

stagiaires:
  - stagiaire01
  - stagiaire02
⚠ Règle

Le nom du fichier dans group_vars/ doit être exactement identique au nom du groupe. Une erreur de nommage est ignorée silencieusement.

04Playbook
Anatomie complète
---

- name: Nom du play
  hosts: webservers
  become: true
  vars:
    app_port: 8080

  tasks:

    - name: Installer nginx
      ansible.builtin.apt:
        name: nginx
        state: present          # present / absent / latest
        update_cache: true

    - name: Déployer la config
      ansible.builtin.template:
        src: templates/nginx.conf.j2
        dest: /etc/nginx/nginx.conf
      notify: Restart nginx     # déclenche le handler si changed

  handlers:

    - name: Restart nginx
      ansible.builtin.service:
        name: nginx
        state: restarted
Commandes d'exécution
ansible-playbook -i inventory/hosts.ini site.yml           # exécution
ansible-playbook -i inventory/hosts.ini site.yml --check   # dry-run
ansible-playbook -i inventory/hosts.ini site.yml -v        # verbeux
ansible-playbook -i inventory/hosts.ini site.yml --limit machine2
ansible-playbook -i inventory/hosts.ini site.yml --tags install
ansible-playbook -i inventory/hosts.ini site.yml --ask-become-pass
Statuts de sortie
StatutSignification
okRien modifié, état déjà conforme
changedModification effectuée
skippedTâche ignorée (when non rempli)
unreachableMachine non joignable
failedErreur
05Modules essentiels
ansible.builtin.apt
- ansible.builtin.apt:
    name: nginx           # ou "{{ item }}" en boucle
    state: present        # present / absent / latest
    update_cache: true
ansible.builtin.service / systemd
- ansible.builtin.service:
    name: nginx
    state: started        # started / stopped / restarted / reloaded
    enabled: true
ansible.builtin.user
- ansible.builtin.user:
    name: deployer
    shell: /bin/bash
    create_home: true
    groups: sudo
    state: present
ansible.builtin.copy
- ansible.builtin.copy:
    src: files/app.conf           # fichier statique
    dest: /etc/app/app.conf
    owner: root
    mode: '0644'

# Écrire du texte directement (sans fichier source)
- ansible.builtin.copy:
    content: "setupuser ALL=(ALL) NOPASSWD: ALL\n"
    dest: /etc/sudoers.d/setupuser
    mode: '0440'
    validate: /usr/sbin/visudo -cf %s
ansible.builtin.template
- ansible.builtin.template:
    src: templates/nginx.conf.j2  # ou juste nginx.conf.j2 dans un role
    dest: /etc/nginx/nginx.conf
    mode: '0644'
  notify: Restart nginx
ansible.builtin.file
- ansible.builtin.file:
    path: /opt/myapp
    state: directory              # directory / file / absent / link
    owner: deployer
    mode: '0755'
ansible.builtin.lineinfile
- ansible.builtin.lineinfile:
    path: /etc/ssh/sshd_config
    regexp: '^#?PermitRootLogin'  # ^#? gère ligne commentée et non commentée
    line: 'PermitRootLogin no'
  notify: Restart sshd
ansible.builtin.command / shell
# command : préféré — pas de shell, pas d'injection
- ansible.builtin.command:
    cmd: /usr/bin/myapp --init
    creates: /opt/myapp/.initialized   # idempotence manuelle

# shell : uniquement si pipes |, redirections >, variables env
- ansible.builtin.shell: ps aux | wc -l
  register: nb_processus
Boucles avec loop
# Boucle avec liste directe
- ansible.builtin.apt:
    name: "{{ item }}"    # guillemets obligatoires
    state: present
  loop:
    - vim
    - curl
    - git

# Boucle avec variable
  loop: "{{ outils_base }}"
Conditionnels et register
# Conditionnel
- ansible.builtin.apt:
    name: nginx
    state: present
  when: ansible_os_family == "Debian"

# Capturer le résultat
- ansible.builtin.command: which docker
  register: docker_check
  ignore_errors: true

- ansible.builtin.debug:
    msg: "Docker : {{ 'présent' if docker_check.rc == 0 else 'absent' }}"
06Variables et Jinja2
Priorité des variables (croissante)
PrioritéSource
1 (basse)group_vars/all.yml
2group_vars/nom_groupe.yml
3host_vars/nom_machine.yml
4vars: dans le playbook
5 (haute)--extra-vars en ligne de commande
Syntaxe Jinja2
{{ ma_variable }}                    # afficher une variable
{{ ansible_hostname }}               # afficher un fact
{{ ma_variable | default('none') }} # valeur par défaut
{{ ma_variable | upper }}           # filtre : majuscules
{{ ma_liste | length }}             # filtre : longueur

{% if ssl_enabled %}                 # conditionnel
listen 443 ssl;
{% endif %}

{% for s in stagiaires %}           # boucle
  <li>{{ s }}</li>
{% endfor %}
Ansible Vault — chiffrer les secrets
ansible-vault create secrets.yml          # créer un fichier chiffré
ansible-vault edit secrets.yml            # éditer
ansible-vault encrypt_string 'monMDP' --name 'db_password'

# Lancer avec le vault
ansible-playbook site.yml --ask-vault-pass
ansible-playbook site.yml --vault-password-file .vault_pass
⚠ Jamais en clair dans Git

Ne jamais commiter de mots de passe dans un playbook. Utilise Ansible Vault pour chiffrer les valeurs sensibles.

07Handlers
ℹ Règles des handlers
  • Ne s'exécutent QUE si la tâche notifiante produit un changed
  • S'exécutent une seule fois en fin de play dans la section RUNNING HANDLERS
  • Le nom du notify doit être strictement identique au name du handler
  tasks:
    - name: Déployer la config
      ansible.builtin.template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf
      notify: Redémarrer nginx      # ← nom exact

  handlers:
    - name: Redémarrer nginx        # ← doit correspondre exactement
      ansible.builtin.service:
        name: nginx
        state: restarted
08Roles
Structure d'un role
roles/
└── nginx/
    ├── tasks/main.yml      # tâches (obligatoire)
    ├── handlers/main.yml   # handlers
    ├── templates/*.j2      # templates Jinja2
    ├── files/              # fichiers statiques
    ├── vars/main.yml       # variables (priorité haute)
    └── defaults/main.yml   # variables par défaut (priorité basse)
Contenu d'un tasks/main.yml de role
# Pas d'entête de play — on commence directement par les tâches
- name: Installer nginx
  ansible.builtin.apt:
    name: nginx
    state: present

- name: Déployer la config
  ansible.builtin.template:
    src: nginx.conf.j2      # relatif à roles/nginx/templates/
    dest: /etc/nginx/nginx.conf
  notify: Redémarrer nginx
Appeler les roles dans site.yml
---
- name: Mon play
  hosts: webservers
  become: true
  roles:
    - role: common
      tags: common
    - role: nginx
      tags: nginx
    - role: web
      when: deploy_web | default(true)
Ansible Galaxy
ansible-galaxy install geerlingguy.nginx    # installer un role
ansible-galaxy role init mon_role           # créer la structure
ansible-galaxy list                         # lister les roles

10Erreurs fréquentes à éviter
❌ Mauvais
name: {{ var }}
✅ Correct
name: "{{ var }}"
❌ Mauvais
Tabulations dans YAML
✅ Correct
Toujours des espaces (2 par niveau)
❌ Mauvais
ansible_default_ipv4.adress
✅ Correct
ansible_default_ipv4.address (2 d)
❌ Mauvais
Mots de passe en clair dans les vars
✅ Correct
Ansible Vault pour tous les secrets
❌ Mauvais
Lancer sans --check en production
✅ Correct
Toujours tester avec --check d'abord
❌ Mauvais
Tout dans un seul playbook monolithique
✅ Correct
Découper en roles réutilisables
❌ Mauvais
Valeurs en dur dans le playbook
✅ Correct
Variables dans group_vars/ ou host_vars/
❌ Mauvais
shell: quand command: suffit
✅ Correct
command: sauf si pipes/redirections nécessaires
